# @提及功能快速使用指南

## 🚀 快速开始

### 1. 访问 WebUI
打开浏览器访问: http://127.0.0.1:8765

### 2. 切换到实时聊天
点击左侧导航栏的 **🔴 实时聊天**

### 3. 选择频道
从顶部下拉框选择一个频道（如 `general`）

---

## 💬 三种 @提及方式

### 方式 1: @All - 发送给所有人

```
步骤:
1. 点击绿色的 "@All" 按钮
2. 输入消息内容
3. 点击发送

效果: 所有成员都会收到通知
```

**示例:**
```
发送给: [@All] ✓
消息: "请大家注意，系统即将维护"
```

---

### 方式 2: 点击成员按钮 - 选择特定成员

```
步骤:
1. 看到成员列表按钮: @seller-fish, @buyer-fish 等
2. 点击要发送的成员（可多选）
3. 输入消息内容
4. 点击发送

效果: 只有选中的成员会收到通知
```

**示例:**
```
发送给: [@seller-fish] ✓ [@buyer-fish] 
消息: "你们可以开始讨价还价了"
```

**提示:**
- 再次点击已选中的按钮可以取消
- 点击 @All 会自动选中所有成员

---

### 方式 3: 输入框内 @自动补全 ⭐推荐

```
步骤:
1. 在输入框中输入 @
2. 自动弹出成员列表
3. 继续输入名字过滤（如 @sel）
4. 使用 ↑↓ 箭头选择，或鼠标点击
5. 按 Enter 或 Tab 确认
6. 继续输入消息内容
7. 点击发送

效果: 智能补全并高亮显示
```

**示例:**
```
输入: @sel → 自动补全为 @seller-fish 
消息: "@seller-fish 请报价"
```

**快捷键:**
- `↑↓` - 上下选择候选项
- `Enter/Tab` - 确认选择
- `Escape` - 关闭补全框

## 🎯 实际使用场景

### 场景 1: 一对一沟通
```
1. 输入 @sel
2. 选择 @seller-fish
3. 输入 "今天的鱼新鲜吗？"
4. 发送

结果: 只有 seller-fish 收到并回复
```

### 场景 2: 小组讨论
```
1. 点击 @seller-fish 按钮
2. 点击 @buyer-fish 按钮
3. 输入 "你们商量一下价格"
4. 发送

结果: seller-fish 和 buyer-fish 都参与讨论
```

### 场景 3: 全员通知
```
1. 点击 @All 按钮
2. 输入 "会议将在 5 分钟后开始"
3. 发送

结果: 所有成员都收到通知
```

### 场景 4: 混合提及
```
1. 输入 @all 触发补全，选择 @All
2. 继续输入 "@seller-fish 请准备报告"
3. 发送

结果: 所有人都看到消息，特别提及 seller-fish
```

无需手动输入 @名字

## ⚠️ 注意事项

### 1. 提及提取规则
- 按钮选中的成员会自动添加
- 输入框中的 @名字也会被识别
- 两者会合并，不会重复

### 2. 空频道处理
- 如果频道没有成员，不会显示按钮
- 需要先添加成员到频道

### 3. 消息类型
- 所有提及消息都是 `mention` 类型
- Worker 会根据 mentions 字段判断是否响应

---

## 🔧 故障排除

### 问题 1: 看不到成员按钮
**原因**: 频道还没有成员  
**解决**: 先在频道管理中添加成员

### 问题 2: 自动补全不出现
**原因**: @后面有空格或特殊字符  
**解决**: 确保 @紧跟字母，如 @seller 不是 @ seller

### 问题 3: 发送后没反应
**原因**: Worker 可能没有订阅该频道  
**解决**: 检查 Worker 的订阅设置
