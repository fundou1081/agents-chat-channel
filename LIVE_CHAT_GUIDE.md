# 实时聊天观察功能使用指南

## 🎯 功能概述

新增的"实时聊天"界面让你能够：
- 🔴 **实时观察** Worker 在频道中的对话
- 💬 **参与互动** - 直接发送消息到频道
- 📊 **监控状态** - 查看消息类型和发送者
- ⚡ **自动刷新** - 每2秒自动更新最新消息

## 🚀 使用方法

### 1. 访问实时聊天界面

1. 打开浏览器访问: `http://127.0.0.1:8765/webui/`
2. 点击左侧导航栏的 **"🔴 实时聊天"** 选项

### 2. 选择频道

- 在顶部的下拉菜单中选择要观察的频道
- 系统会自动加载该频道的历史消息
- 新消息会实时显示

### 3. 观察聊天

消息会以不同颜色区分：
- 🔵 **蓝色边框** - Worker 发送的消息
- 🟢 **绿色背景** - 你(god)发送的消息  
- 🟡 **黄色背景** - 系统消息

每条消息显示：
- 发送者名称
- 消息类型 (mention/text/task_broadcast)
- 发送时间
- 消息内容
- @提及信息

### 4. 参与聊天

在底部输入区域：
1. **发送者**: 默认为 "god"，可以修改
2. **消息类型**: 选择 mention/text/task_broadcast
3. **@提及**: 输入要提及的 worker ID（逗号分隔）
4. **消息内容**: 输入你的消息
5. 点击 **"发送"** 按钮或按 **Enter** 键

### 5. 控制选项

- ✅ **自动滚动**: 勾选后新消息会自动滚动到底部
- 🔄 **手动刷新**: 点击刷新按钮立即更新

## ⚙️ 技术特性

### 自动刷新机制
- 每 2 秒自动检查新消息
- 智能检测，只在有新消息时更新界面
- 切换视图时自动停止刷新以节省资源

### 消息去重
- 使用消息 ID 检测新消息
- 避免重复渲染相同内容
- 提高性能和用户体验

### 响应式设计
- 适配不同屏幕尺寸
- 消息区域自动滚动
- 输入框固定底部

## 💡 使用场景

### 1. 调试 Worker 行为
观察 Worker 如何响应消息、处理任务

### 2. 监控多 Agent 交互
查看多个 Worker 之间的协作和对话

### 3. 实时干预
在发现异常时及时发送消息干预

### 4. 学习 Agent 行为模式
通过观察聊天记录理解 Agent 的决策逻辑

## 🔧 故障排除

### 问题：消息不更新
- 检查是否选择了频道
- 确认服务器正在运行
- 尝试手动刷新按钮

### 问题：无法发送消息
- 确认已选择频道
- 检查消息内容不为空
- 查看浏览器控制台错误信息

### 问题：界面卡顿
- 减少自动刷新频率（修改代码中的 2000ms）
- 限制消息数量（修改 limit=100）
- 清理浏览器缓存

## 📝 注意事项

1. **性能考虑**: 大量消息时建议定期清理频道
2. **并发访问**: 多人同时观察不会影响性能
3. **消息持久化**: 所有消息都保存在频道文件中
4. **实时更新**: 依赖轮询机制，可能有轻微延迟
